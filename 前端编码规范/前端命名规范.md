# 前端命名规范

## 目录命名

每个项目下的目录

```
- 简洁明了(如下：)
├─ src          // 源代码
│  ├─ components// 组件资源
│  ├─ mock      // 本地mock
│  ├─ static    // 静态资源
│  │  ├─ images // 图片
├─ doc          // 文档
├─ test         // 测试用例
```

## 函数命名规范

命名方法： 小驼峰式命名法

命名规范： 前缀应该为动词

命名建议： 常用动词约定

| 词语           | 含义                   |
| -------------- | ---------------------- |
| can            | 判断是否可执行某个动作 |
| has            | 判断是否含有某个值     |
| is             | 判断是否为某个值       |
| get            | 获取某个值             |
| set            | 设置某个值             |
| load           | 加载某些数据           |
| handle（后缀） | 某个事件的执行回调     |
| check          | 执行检测操作           |
